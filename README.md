## 🛡️ Security-First Engineering

While most developers treat security as a post-deployment checklist, I **bake it into the architecture from day one**.

| Phase | Practice |
| :--- | :--- |
| **Threat Modeling** | Identify risks early in the design phase |
| **Secure Design** | OWASP-aligned architecture with defense-in-depth |
| **Input Validation** | Zod schemas on every API endpoint and form |
| **Auth Hardening** | JWT + HttpOnly cookies + RBAC + session management |
| **Rate Limiting** | DDoS protection on all public endpoints |
| **Security Headers** | Helmet.js configuration for XSS, CSRF, clickjacking protection |
| **Automated Scanning** | CI/CD pipeline with vulnerability checks |
| **Monitoring** | Sentry error tracking + structured logging |

**Result:** Applications that are **secure by default**, not secure by accident.
